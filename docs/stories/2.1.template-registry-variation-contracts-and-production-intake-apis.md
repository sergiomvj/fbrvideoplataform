## Story

**As a** production operator,  
**I want** the system to know which templates and variations exist and to accept new productions against those contracts,  
**so that** every production starts with a valid, constrained and explicit operational setup.

---

## Acceptance Criteria

1. The system exposes a template registry containing the two approved templates and their three variations each.
2. Each template contract includes format, duration limits, mode compatibility and composition constraints required by downstream modules.
3. Production intake APIs accept new jobs in automatic or manual mode with explicit template and variation selection.
4. Intake validation rejects invalid template, variation or mode combinations.
5. The created production is persisted in the core workflow state model with enough metadata for later editorial and media stages.
6. Template metadata is consumable by the frontend wizard without extra interpretation logic.
