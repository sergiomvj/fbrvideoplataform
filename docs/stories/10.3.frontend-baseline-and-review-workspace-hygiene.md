## Story

**As a** frontend maintainer,  
**I want** the frontend baseline and review workspace cleaned up to match the documented shell expectations,  
**so that** Story 5.1 is genuinely compliant and the operator experience is not carrying preventable technical debt.

---

## Acceptance Criteria

1. `npm run lint` passes for the frontend with no errors.
2. The review workspace no longer contains React effect anti-patterns that fail the agreed lint rules.
3. Shared UI primitives and review components meet the current frontend quality baseline.
4. Navigation and screen-level expectations defined in Story 5.1 are either implemented or explicitly constrained/documented with no misleading dead paths.
5. Story 5.1 can be re-reviewed without baseline hygiene objections.
