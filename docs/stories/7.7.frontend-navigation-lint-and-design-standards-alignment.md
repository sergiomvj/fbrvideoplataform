## Story

**As a** product owner,  
**I want** the shell, navigation and base frontend implementation to match the real product and approved standards,  
**so that** the console feels coherent and the frontend baseline is safe for continued development.

---

## Acceptance Criteria

1. `npm run lint` passes.
2. The shell does not point users to routes that do not exist.
3. Base layout typography and metadata are aligned with approved design standards or explicitly documented as an exception.
4. Unused code and structural lint debt in the reviewed frontend scope are removed.
5. Navigation reflects the actual MVP surface.
