## Story

**As a** product team,  
**I want** the frontend shell and design-system foundation established early,  
**so that** all later UI stories can be implemented consistently and without one-off interface drift.

---

## Acceptance Criteria

1. The frontend app has a persistent shell structure aligned with the UX/interface document.
2. Base typography, colors, spacing and reusable primitives are configured according to `DESIGN_STANDARDS.md`.
3. Navigation scaffolding exists for the main console areas.
4. The design-system foundation supports the operational states required later (success, warning, danger, info).
5. Accessibility basics are present from the start.
