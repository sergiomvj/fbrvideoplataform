## Story

**As a** maintainer and operator,  
**I want** technical documentation and functional help aligned with the corrected system behavior,  
**so that** developers onboard faster and operators trust what the system claims to do.

---

## Acceptance Criteria

1. `frontend/README.md` is replaced with real module documentation.
2. `backend/README.md` exists and documents the backend clearly.
3. `docs/help-completo-funcionalidades.md` reflects the corrected product state.
4. Documentation distinguishes current functionality from future planned functionality.
5. Key operational flows are traceable from README/help to the code and stories.
