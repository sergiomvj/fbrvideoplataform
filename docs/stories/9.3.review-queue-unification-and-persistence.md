## Story

**As a** media workflow owner,  
**I want** a single authoritative review queue implementation backed by persistence,  
**so that** requery, human review and queue actions behave consistently across backend and UI.

---

## Acceptance Criteria

1. The system uses one authoritative review queue model only.
2. Queue items are persisted with score, rationale, scene context and candidate context.
3. Requery, review-required and structural-failure paths integrate with the central workflow consistently.
4. Queue actions are persisted and auditable.
5. The review API exposes enough information for operator action without frontend-side invention.
6. The system no longer depends on ephemeral in-memory review queue state.
