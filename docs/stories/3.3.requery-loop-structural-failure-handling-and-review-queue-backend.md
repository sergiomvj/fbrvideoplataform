## Story

**As a** media workflow controller,  
**I want** low-confidence candidates to go through controlled requery and only plausible candidates to reach review,  
**so that** human attention is spent well and the pipeline remains operationally disciplined.

---

## Acceptance Criteria

1. Candidates below `60%` trigger requery behavior instead of human review.
2. Candidates between `60–89%` enter a review queue with all required inspection data.
3. The system detects retry exhaustion and marks structural failure explicitly.
4. Review queue items are persisted with enough information for operator action.
5. Requery, review-required and structural-failure states integrate with the central production workflow model.

### Technical Notes

- Avoid mixing UI concerns into the queue model.
- Retry counts and exhaustion rules must be explicit and queryable.
