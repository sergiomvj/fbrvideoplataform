## Story

**As a** platform operator,  
**I want** a persistent registry of image and video providers with operational configuration,  
**so that** the system can manage, enable, disable and audit the external sources used for media acquisition.

---

## Acceptance Criteria

1. The system supports persistent registration of media providers for image and video sourcing.
2. A provider record includes at least: provider key, provider type, source type, status, auth/config payload and operational metadata.
3. Providers can be enabled or disabled without code edits.
4. The media sourcing layer resolves active providers from persistence/configuration rather than hardcoded startup-only registration.
5. Provider usage is traceable through integration and sourcing logs.
6. The provider registry contract is documented for future operations and admin UI work.
