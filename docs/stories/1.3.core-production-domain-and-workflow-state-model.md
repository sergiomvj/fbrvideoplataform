## Story

**As a** platform architect,  
**I want** a formal production domain model with explicit workflow states and transitions,  
**so that** all modules can integrate through a shared and predictable lifecycle instead of ad hoc orchestration.

---

## Acceptance Criteria

1. A central production aggregate exists with identifiers, mode, template, variation, lifecycle state and audit timestamps.
2. Workflow states cover the full high-level lifecycle from intake through structuring, media evaluation, review, composition, render and completion/failure.
3. State transitions are explicit and validated rather than inferred implicitly by route handlers or UI logic.
4. The model supports both automatic-assisted and manual-guided modes without branching into incompatible lifecycles.
5. Domain events or equivalent state-change records are available for observability and orchestration consumers.
6. The resulting workflow model is documented for all downstream stories to consume.

### Technical Notes

- Keep state logic in the domain/application layer, not in UI or integrations.
- Explicitly reserve states for review, requery and structural failure.
- Downstream stories will rely on this contract heavily, so stability is more important than breadth.
