## Story

**As a** render pipeline owner,  
**I want** Story 8.2 completed through a real canonical submission flow wired into the API surface,  
**so that** external video execution is reliable, persisted and consistent with the planned architecture.

---

## Acceptance Criteria

1. Render submission routes use the canonical submission service/contract instead of calling provider adapters directly.
2. Submission validation, provider submission, polling/webhook reconciliation and terminal-state handling are part of one coherent operational flow.
3. Submission failures, validation failures and provider failures are persisted and distinguishable.
4. Render lifecycle APIs expose the backend-authoritative state derived from persisted submission/render truth.
5. Story 8.2 can be re-reviewed as compliant against its original acceptance criteria.
