## Story

**As a** render orchestration owner,  
**I want** compositions, render jobs and render events persisted with clean provider isolation,  
**so that** render monitoring survives restarts and external failures remain contained.

---

## Acceptance Criteria

1. The system no longer depends on `_composition_store`.
2. The system no longer depends on in-memory `RenderJobStore` for authoritative lifecycle state.
3. Compositions are persisted and queryable for UI and provider submission.
4. Render jobs and render events are persisted and queryable.
5. Provider submission failures are isolated from composition state corruption.
6. Render status APIs return persisted lifecycle truth.
