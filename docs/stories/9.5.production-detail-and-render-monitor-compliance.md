## Story

**As an** operator,  
**I want** the production detail screen to match the real backend contract for workflow, composition and render,  
**so that** I can trust what I see without hidden gaps or invented frontend fields.

---

## Acceptance Criteria

1. A canonical production detail payload is defined and implemented.
2. The payload includes current state, state history, template/variation identity, composition preview and render status.
3. The frontend detail screen consumes only canonical payload fields.
4. Failure, blocked and empty states are represented clearly.
5. The render monitor reflects persisted render jobs and statuses.
6. `npm run lint` passes for the detail screen scope.
