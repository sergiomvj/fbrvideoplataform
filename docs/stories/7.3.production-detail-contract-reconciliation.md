## Story

**As an** operator,  
**I want** the production detail screen to reflect the real backend contract,  
**so that** I can trust the current state, composition preview and render status of each production.

---

## Acceptance Criteria

1. A canonical production detail payload is defined and implemented.
2. The payload includes current state, state history, template/variation identity, composition preview and render status.
3. The frontend detail page consumes only fields actually provided by the backend.
4. The UI no longer infers missing structure from guessed property names.
5. The detail contract is documented for future screens and integrations.
