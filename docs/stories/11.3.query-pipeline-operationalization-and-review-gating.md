## Story

**As a** media intelligence owner,  
**I want** the live sourcing and routing flow to use the hardened query pipeline,  
**so that** contextual search, reformulation and review eligibility behave exactly as documented.

---

## Acceptance Criteria

1. The live sourcing route builds query strategy through `QueryBuilderService` from visual-brief context.
2. Query attempts, diagnostics and reformulation lineage are created through the query-builder flow rather than ad hoc route construction.
3. Results below the plausibility threshold trigger reformulation before review eligibility.
4. Review queue eligibility is enforced through routing logic backed by persistence.
5. No candidate reaches human review without having crossed the documented minimum plausibility threshold.
6. Stories `8.3` and `10.7` become re-reviewable as compliant.
