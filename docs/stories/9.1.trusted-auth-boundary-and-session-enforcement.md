## Story

**As a** platform owner,  
**I want** all authenticated flows to rely on a trusted session boundary,  
**so that** no client can impersonate arbitrary users or reach protected backend routes with forged identity.

---

## Acceptance Criteria

1. The frontend no longer creates authenticated sessions from arbitrary `userId` input.
2. The production wizard no longer performs fake login with a hardcoded operator.
3. The backend no longer trusts raw browser-provided `X-User-Id` as sufficient proof of identity.
4. Protected requests require a valid authenticated session routed through the controlled gateway.
5. Unauthorized, malformed and expired session scenarios return safe `401` or `403` responses without leaking internals.
6. The trusted auth contract is documented and reusable by all secured stories.

---

## Tasks / Subtasks

- [ ] Task 1: Remove fake login/session bootstrap (AC: 1, 2)
  - [ ] Remove hardcoded session creation from the wizard
  - [ ] Prevent arbitrary session creation through the auth route
  - [ ] Ensure anonymous usage fails safely

- [ ] Task 2: Define canonical trusted auth context (AC: 3, 4, 6)
  - [ ] Define the gateway-to-backend auth context shape
  - [ ] Include future-safe fields for `user_id` and `organization_id`
  - [ ] Document trust boundary and propagation rules

- [ ] Task 3: Harden backend auth enforcement (AC: 3, 4, 5)
  - [ ] Refactor protected-route auth dependency
  - [ ] Reject malformed or untrusted context
  - [ ] Standardize sanitized auth failures

- [ ] Task 4: Add verification coverage (AC: 4, 5, 6)
  - [ ] Session success/failure tests
  - [ ] Protected route authorization tests
  - [ ] Gateway propagation tests
