## Story

**As a** data architect,  
**I want** an initial PostgreSQL schema derived from the approved domain model,  
**so that** the system can migrate off memory stores safely and preserve complete operational history.

---

## Acceptance Criteria

1. An initial SQL schema exists for the approved production, review, composition, render and audit domains.
2. The schema defines primary keys, foreign keys, core constraints and essential indexes.
3. The schema separates current-state tables from append-only historical tables where appropriate.
4. The schema is organized so that remediation persistence stories can adopt it incrementally.
5. The schema is documented sufficiently for migration planning and repository implementation.
