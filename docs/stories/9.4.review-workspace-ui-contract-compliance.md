## Story

**As a** production operator,  
**I want** the review workspace to be fully aligned with the backend contract,  
**so that** I can inspect, approve, reject and requery candidates without broken data or invalid routes.

---

## Acceptance Criteria

1. The review page consumes only supported review endpoints.
2. The workspace renders scene label, scene index, candidate preview, score, rationale and status correctly.
3. Approve, reject and requery actions call valid backend routes and update UI state correctly.
4. Auto-approved, review-required, manual-fixed and rejected states are visually distinct where relevant.
5. The page works with both empty queue and scoped production review cases.
6. `npm run lint` passes for the review workspace scope.
