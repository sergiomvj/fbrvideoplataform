## Acceptance Criteria

1. `frontend/README.md` documents the real frontend module instead of boilerplate.
2. `backend/README.md` exists and documents the real backend module.
3. `docs/help-completo-funcionalidades.md` reflects only true, current system behavior after remediation.
4. Key docs are cross-traceable to the corrected stories and code paths.
5. The remediation closeout includes evidence for `lint`, `build` and backend test execution requirements.
6. The system’s corrected behavior is documented clearly enough for developer onboarding and operator usage.
