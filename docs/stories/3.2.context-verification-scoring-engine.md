## Story

**As a** context verification subsystem,  
**I want** to score candidate assets against scene briefs with explicit rationale,  
**so that** the system can automate only when contextual confidence is high and escalate correctly otherwise.

---

## Acceptance Criteria

1. Candidate assets can be scored against visual briefs using the minimum criteria defined in the SPEC.
2. The result includes score, rationale, failure flags and operational status suggestion.
3. The engine supports the three operational bands: `90–100`, `60–89`, `<60`.
4. The engine is explainable enough for UI inspection and audit records.
5. The output is consumable by review queue, requery loop and composition gating.

### Technical Notes

- Final score weights can evolve, but the result contract and thresholds should be stable.
- This story should not implement the requery loop itself.
- Prefer explicit outputs over hidden scoring state.
