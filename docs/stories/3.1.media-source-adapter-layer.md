## Story

**As a** media sourcing subsystem,  
**I want** a unified adapter layer for retrieving candidate assets from automatic and controlled sources,  
**so that** the rest of the system can source media without depending on provider-specific implementations.

---

## Acceptance Criteria

1. A common adapter contract exists for retrieving candidate assets from supported sources.
2. The adapter layer consumes structured visual briefs as input.
3. Automatic external sources and controlled/manual/internal sources can be represented behind the same candidate contract.
4. Provider-specific failures are isolated and returned as controlled integration outcomes.
5. The output is consumable by the context-verification engine without provider-specific logic leaking upward.
