## Story

**As a** platform operator,  
**I want** provider persistence to be the real runtime source of truth,  
**so that** provider enablement, disablement and sourcing behavior are stable, governable and restart-safe.

---

## Acceptance Criteria

1. `ProviderRegistryService` is wired to `ProviderRepository` in runtime, not only in code structure.
2. Backend startup initializes provider state from PostgreSQL and only seeds defaults through persistence-safe flow.
3. `MediaSourcingService` resolves active providers through the runtime service/repository path rather than the bare global registry.
4. Backend exposes operational provider-registry endpoints for create/list/update/enable/disable/delete.
5. Restarting the backend preserves provider behavior and enabled/disabled state without manual re-registration.
6. Story `8.1` and `10.5` become re-reviewable as compliant.
