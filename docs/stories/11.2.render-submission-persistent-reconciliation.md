## Story

**As a** render pipeline owner,  
**I want** submission and reconciliation state to come from persisted lifecycle records,  
**so that** render monitoring survives restart and remains authoritative for backend, frontend and orchestration.

---

## Acceptance Criteria

1. `RenderSubmissionService` is wired to `RenderSubmissionRepository` in runtime.
2. Submission lookup and production-level submission history can be recovered from persistence after restart.
3. Reconciliation reads persisted submissions instead of only the in-memory dict.
4. Webhook handling is exposed through API and updates persisted submission state.
5. Submission, validation and provider failure categories remain distinguishable in persistence and APIs.
6. Stories `8.2` and `10.6` become re-reviewable as compliant.
