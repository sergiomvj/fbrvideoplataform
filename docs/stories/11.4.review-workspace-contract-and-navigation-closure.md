## Story

**As a** production operator,  
**I want** the review workspace and its backend contract to expose real review context and valid navigation,  
**so that** the review desk behaves like a trustworthy operational screen instead of a placeholder shell.

---

## Acceptance Criteria

1. Persisted review items include the contextual fields the UI expects: scene label/index, score, rationale, candidate/source metadata and preview information.
2. The review API returns real persisted values rather than synthetic fallback defaults wherever the data should exist.
3. The review screen entry path is valid from shell navigation and does not drop users into a missing `production_id` state without guidance.
4. Approve/reject/requery actions operate on persisted review truth and return updated state consistently.
5. Story `5.3` becomes re-reviewable as compliant.
