## Acceptance Criteria

1. The repository contains concrete n8n integration artifacts or documented workflow bootstrap assets matching Story 6.1 expectations.
2. The event surface consumed by orchestration is documented with payload examples and trigger semantics.
3. OpenClaw and FBR-Click integration hooks remain outside domain logic and are clearly identified as contracts or runtime adapters.
4. Failure and escalation paths expected by Story 6.1 are represented in the runtime-facing documentation/assets.
5. Story 6.1 can be re-reviewed as compliant without relying on missing directories or implicit future work.
