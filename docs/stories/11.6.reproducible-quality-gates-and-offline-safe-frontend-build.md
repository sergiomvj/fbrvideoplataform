## Story

**As a** delivery owner,  
**I want** the declared quality gates to be reproducible in constrained local QA environments,  
**so that** build/typecheck/test results actually mean the project is healthy.

---

## Acceptance Criteria

1. `npm run lint` remains green from the repository root.
2. `npm run typecheck` or its approved equivalent passes without depending on live Google Fonts fetches.
3. Frontend typography/build configuration is offline-safe or otherwise reproducible in the expected local QA environment.
4. `npm test` is either backed by real checks or the gate contract is explicitly updated so it no longer pretends to validate what does not exist.
5. The documented root quality gates in `AGENTS.md` match the actual runnable project setup.

---

## Tasks / Subtasks

- [ ] Task 1: Make frontend build reproducible (AC: 2, 3)
  - [ ] Remove runtime dependence on remote font fetch for local build
  - [ ] Switch to local/vendorized fonts or documented approved equivalent
  - [ ] Revalidate root build/typecheck gate

- [ ] Task 2: Reconcile root quality-gate contract (AC: 1, 4, 5)
  - [ ] Audit the meaning of root `lint`, `typecheck`, and `test`
  - [ ] Replace placeholder `test` or downgrade the contract transparently
  - [ ] Align `AGENTS.md` with the true runnable commands

- [ ] Task 3: Reclose compliance evidence (AC: 5)
  - [ ] Document the final gate behavior
  - [ ] Ensure QA can rerun the declared gates reliably
