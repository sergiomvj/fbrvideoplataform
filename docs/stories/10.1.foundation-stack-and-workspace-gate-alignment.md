## Story

**As a** platform maintainer,  
**I want** the workspace stack, version declarations and quality-gate commands aligned with the documented foundation,  
**so that** Story 1.1 can be considered fully compliant and future stories build on a trustworthy baseline.

---

## Acceptance Criteria

1. The frontend runtime version strategy is reconciled with the canonical foundation decision for Story 1.1, either by aligning the implementation or by updating the architecture/story artifacts through an explicit approved decision.
2. The repository exposes executable quality-gate commands matching `AGENTS.md`, or `AGENTS.md` is updated to reflect the true command surface with no dead root-level commands.
3. Foundation documentation clearly states the authoritative workspace entrypoints for frontend, backend and infra validation.
4. Any stack/version exception from the original story is documented as a deliberate decision rather than accidental drift.
5. Story 1.1 can be re-reviewed without ambiguity about stack compliance.
