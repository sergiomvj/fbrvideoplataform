## Story

**As an** editorial operator,  
**I want** the review queue UI and API to speak the same contract,  
**so that** I can inspect candidates, understand their scores, and approve/reject/requery safely.

---

## Acceptance Criteria

1. The backend review endpoints return all fields needed by the review UI: scene label, scene index, score, justification, preview URL and status.
2. The frontend review workspace only calls supported endpoints.
3. `approve`, `reject` and `requery` have explicit endpoint contracts and state semantics.
4. The review page can operate both scoped to a production and through a valid queue retrieval mode.
5. The data contract is documented for reuse by future persistence and analytics work.
