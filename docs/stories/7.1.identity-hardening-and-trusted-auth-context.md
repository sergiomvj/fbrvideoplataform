## Story

**As a** platform operator and system owner,  
**I want** authenticated requests to carry trusted identity only,  
**so that** no client can impersonate arbitrary users or bypass protected backend operations.

---

## Acceptance Criteria

1. The frontend no longer creates authenticated sessions from arbitrary `userId` payloads.
2. The wizard flow does not bootstrap login with a hardcoded operator such as `current-user`.
3. The backend no longer accepts identity based solely on a user-controlled `X-User-Id` without a trusted gateway contract.
4. Protected requests without valid session/context return `401` consistently.
5. The authenticated context contract explicitly supports `user_id` and prepares for `organization_id`.
6. Security behavior is documented and reusable by future frontend/backend stories.
